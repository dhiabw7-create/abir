# MedFlow Clinic SaaS

Production-grade multi-tenant clinic management platform inspired by MEDWIN-style workflows, built with:

- Frontend: React 18 + Vite + TypeScript + Tailwind + Radix + Framer Motion + Recharts
- Backend: NestJS + TypeScript + Prisma
- Database: MySQL 8
- Realtime: Socket.IO (waiting room + notifications)
- Queue/Cache: Redis + BullMQ
- Storage: local dev + S3-compatible (MinIO)
- Monorepo: npm workspaces

Medical disclaimer: **This software does not provide medical advice.**

## Monorepo Structure

```text
/apps
  /web        React app (routes, UI, pages)
  /api        NestJS API (modules, guards, sockets, jobs)
/packages
  /ui         Shared UI components
  /shared     Shared TS types + Zod schemas
```

## Roles

- `SUPER_ADMIN`: platform settings, tenants, plans, global templates, catalog import, backups, audit.
- `DOCTOR`: full medical workflow (patients, consultations, prescriptions, CNAM, docs, waiting room actions).
- `SECRETARY`: agenda/check-in/registration/upload admin docs, with restricted medical visibility.

## Multi-Tenant

- All domain records include `tenantId`.
- Tenant isolation enforced via `TenantGuard`.
- Super admin can switch active tenant by setting `x-tenant-id` (UI switcher included).
- Audit logging enabled for auth + CRUD mutations.

## Main Modules Implemented

- Auth + JWT access/refresh rotation + httpOnly refresh cookie
- RBAC guards (frontend + backend)
- Patients + advanced search + timeline views
- Agenda + appointments states + reminders
- Realtime waiting room (Socket.IO)
- Consultations + vitals + diagnosis placeholder + quick templates
- Prescriptions + medication catalog + DEMO interactions + PDF export
- Document templates (WYSIWYG-like placeholder editor) + generated PDFs
- CNAM workflows (verification, bordereaux, TXT export, archive, corrective flow, plafond, carnets)
- Finance dashboards + CSV exports
- Reports + activity logs
- Settings (tenant data, appearance, language scaffold FR/AR, working hours)
- Admin workspace (tenants, users, import wizard, templates, audits, backups)
- Daily scheduled + manual backups

## Local Development (without Docker)

1. Install dependencies:

```bash
npm install
```

2. Set API `DATABASE_URL` and run migration SQL against MySQL, or start Docker MySQL first.
3. For local mode without Redis, set `REDIS_DISABLED=true` in `apps/api/.env`.

4. Generate Prisma client:

```bash
npm run prisma:generate -w @medflow/api
```

5. Apply database migrations:

```bash
npm run prisma:deploy -w @medflow/api
```

6. Seed demo data:

```bash
npm run prisma:seed -w @medflow/api
```

7. Run apps:

```bash
npm run dev
```

- Web: `http://localhost:5173`
- API: `http://localhost:3000`
- Swagger: `http://localhost:3000/api/docs`

## Deployment Notes

- Use managed MySQL + Redis in production.
- Set `STORAGE_MODE=s3` and configure MinIO/S3 credentials.
- Put API behind HTTPS reverse proxy; set `COOKIE_SECURE=true`.
- Keep `JWT_ACCESS_SECRET` and `JWT_REFRESH_SECRET` in secret manager.
- Enable scheduled backups and ship backup artifacts to durable object storage.
- Add WAF/rate-limit and centralized logging/monitoring for production hardening.
